# 文章结构模板

按分类给的起手大纲。用户写作时主导内容，模板只是「免空白页恐惧」用的脚手架，不强制。

## 项目分享

```markdown
## 缘起
[为什么做这个 / 解决什么痛点]

## 怎么做的
[关键技术决策 + 踩坑]

## 效果
[截图 / 数字 / 自己用的体验]

## 给后来者
[文档链接 / 仓库链接 / 待办]
```

适用：`category: 项目分享`。语气可以稍微自夸但不要油腻。

## 技术笔记

```markdown
## 背景
[遇到了什么]

## 探索过程
[尝试了什么 / 哪些不行]

## 解决方案
[最终做法 + 原理]

## 沉淀
[下次遇到类似的怎么办]
```

适用：`category: 技术笔记`。「探索过程」节适合用 `:::fold[实验记录]` 收起来，让主线干净。

## 学习总结

```markdown
## 学了什么
[一句话主旨]

## 关键观点
[3-5 条 takeaway，每条一段]

## 我的延伸思考
[把作者观点和自己经验绑]

## 推荐
[书 / 课程链接，给同好]
```

适用：`category: 学习总结`。引用书原句可以用 `> blockquote`，不要用 `:::info`（语义混淆）。

## 生活随笔

```markdown
## [一个场景开场，不写"## 引言"]
直接进入故事/观察，不要套路化开场。

[多段叙事或观察，可以用小标题切场景，也可以一气呵成]

## [收尾段，不写"## 总结"]
留一个余韵就好，别强行升华。
```

适用：`category: 生活随笔`。标题用具体场景名（如「上海五一两天」「小区门口的猫」），不要抽象的「## 引言/正文/总结」。

## 碎碎念

随意，可以一段到底。**没有强制结构**。

写作建议：
- 用 `:::spoiler` 藏不想被搜索引擎过分抓但想留下的想法
- 用 `:::fold[标题]` 收题外话，让主线短
- 不需要小标题，篇幅短的话直接段落
- 不要写得像「日记体」——「今天/昨天」开头太老套，从内容本身切入

---

**共通建议**：
- 第一段抓人（具体场景 / 反常识结论 / 一个问句），不要用「最近一直在想...」这种软启动
- 长文（> 1500 字）务必加小标题切节，否则视觉劝退
- 收尾留白或抛问题，不要强行总结升华
